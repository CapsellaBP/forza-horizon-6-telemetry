# 调校使用指南

## 工作流

### 1. 赛前准备

1. 启动 `python server.py`
2. 浏览器打开 `web/index.html`，点"启动 HUD"
3. 开一辆要调的车，在直线路段**全油门加速**到断油

> 保持油门 >95%（默认采样阈值）。升档、松油、断油下降段的脏数据会被自动过滤。

### 2. 看曲线

Web 面板的功率/扭矩曲线图会逐渐成形：
- **蓝色线** = 扭矩 (Nm)，左轴
- **红色虚线** = 功率 (kW)，右轴
- **橙色竖线** = 计算的换挡点
- **红色区域** = 断油区
- **灰色竖线** = 当前 RPM

HUD RPM 条上也同步显示：
- **橙色竖线** = 换挡点（峰值功率位置）
- **红色竖线** = 断油上界
- **白色区间** = 功率带（峰值功率的 93% 范围，可调）

### 3. 锁定曲线

采样满意后，点"**锁定曲线**"冻结。比赛时不受脏数据影响。

### 4. 调整换挡

- **激进程度**：负值 → 延后升档（更高转速换挡，适合高转引擎），正值 → 提前升档
- **爆闪触发点**：默认 100% 即在峰值功率处蓝闪。调到 95% 则提前提醒
- 调完看曲线图上换挡线位置，找到最佳点

### 5. 断油分析

踩到断油后，服务端自动检测：
- **断油上界**（红线）：振荡转速峰值的中位数
- **断油下界**：振荡谷值，用于剔除污染数据

Web 面板显示"已检测"标签即表示断油数据就绪。

### 6. 功率带

功率带 = 峰值功率 × 阈值（默认 93%）所覆盖的 RPM 区间。

调整阈值：
- 更高（95%+）：只看最窄的甜点区
- 更低（80%）：看更宽的可用区间

## 电动车

EV 自动识别：全油门时档位 ≤2 持续 N 帧（默认 300）判为电车。

电车上 HUD 不显示换挡线和断油线，RPM 条全程蓝色。

## 换车

换车后：
- 旧车的换挡激进、断油阈值、曲线数据自动保存
- 新车加载已有预设（如有），否则默认值
- 全新车需重新踩全油门采样

Web 面板"重置曲线"按钮清空当前车数据。

## 常见问题

**Q: 功率曲线忽高忽低？**
A: 调高采样油门阈值（默认 95%）和 EMA 权重（1/8 或 1/16），过滤松油打滑数据。

**Q: 换挡点不对？**
A: 确认断油检测已完成（面板显示"已检测"）。断油区功率不参与峰值计算。

**Q: 档位偶尔跳？**
A: 服务端已做防抖（3 帧确认）和 >10 过滤。偶发的 1-2 帧异常不会显示。

**Q: HUD 不显示？**
A: 游戏必须用**无边框窗口模式**。确认 Data Out 开启且端口 5300。
